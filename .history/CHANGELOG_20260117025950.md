- Length control (1-liner, 2-3 sentences, paragraph)
- CTA options (None, Question, Invite, Resource)
- Persona presets (Builder, Designer, PM, Founder, Recruiter)
- Safety filters (noCringe mode)
- "Why it works" tags for each reply
- Web app landing page
- Interactive demo component
- Database schema with Prisma
- PostgreSQL support
- OpenAI GPT-4 integration
- Comprehensive documentation

## Versioning Strategy

We use [Semantic Versioning](https://semver.org/):

- **MAJOR.MINOR.PATCH**
  - MAJOR: Breaking changes
  - MINOR: New features (backward compatible)
  - PATCH: Bug fixes (backward compatible)

Example:
- `0.1.0` → `0.2.0`: New feature (voice profiles)
- `0.2.0` → `0.2.1`: Bug fix
- `0.2.1` → `1.0.0`: Public launch (breaking API changes)

## Release Process

1. Update version in `package.json`
2. Update this CHANGELOG
3. Create git tag: `git tag -a v0.2.0 -m "Release v0.2.0"`
4. Push tag: `git push origin v0.2.0`
5. Create GitHub release with notes
